Capítulo 1

Introdução ao Arduino

Antes de entender o código, precisamos entender quem executa esse código. O Arduino é um pequeno computador dedicado ao controle de dispositivos eletrônicos. Enquanto um computador comum executa programas complexos como navegadores, jogos e editores de texto, o Arduino foi criado para controlar sensores, motores, relés, LEDs e diversos equipamentos eletrônicos.

O que é um microcontrolador?

O componente principal da placa Arduino é um microcontrolador. Ele reúne em um único circuito:

  • Processador (CPU)
  • Memória RAM
  • Memória Flash
  • Entradas e saídas digitais
  • Entradas analógicas
  • Temporizadores
  • Comunicação Serial

Capítulo 4

Entendendo o #define

Depois de importar a biblioteca, encontramos três linhas muito importantes: 


#define TRIG_PIN 7
#define ECHO_PIN 6
#define RELAY_PIN 8

Essas linhas definem quais pinos físicos do Arduino serão utilizados durante o projeto.

O que significa #define?

O comando #define cria uma macro. Antes do programa ser compilado, o pré-processador substitui automaticamente um nome por outro valor. 


TRIG_PIN

↓

7

Sempre que o compilador encontrar TRIG_PIN, ele substituirá pelo número 7.

Exemplo 


digitalWrite(TRIG_PIN, HIGH);

Internamente o compilador transforma em: 


digitalWrite(7, HIGH);

Por que usar nomes?

Imagine um programa com milhares de linhas. Se você escrever apenas o número 7 em vários lugares e depois decidir mudar o sensor para o pino 9, será necessário procurar todos os números manualmente.

Com o #define, basta alterar uma única linha. 


#define TRIG_PIN 9

Cada definição do projeto

Macro Função
TRIG_PIN Pino que envia o pulso ultrassônico.
ECHO_PIN Pino que recebe o eco.
RELAY_PIN Pino que controla o módulo relé.

Ligação física 


Arduino

      Pino 7 -------- TRIG

      Pino 6 -------- ECHO

      Pino 8 -------- RELÉ
Você sabia?

O símbolo "#" indica uma diretiva do pré-processador. Ela é executada antes da compilação do programa.
Erro comum

Muitos iniciantes acreditam que #define cria uma variável. Na realidade ele apenas faz uma substituição de texto.

Resumo

  • Define nomes para valores.
  • Facilita manutenção.
  • Não ocupa memória RAM.
  • É processado antes da compilação.

Capítulo 6

Variáveis, Constantes e Memória do Arduino

Chegamos agora a uma das partes mais importantes de todo o programa. Antes de tomar qualquer decisão, o Arduino precisa armazenar informações. Essas informações ficam dentro da memória RAM do microcontrolador.

O trecho do código 



bool bombaLigada = false;
bool falhaSeguranca = false;

const int LIMIAR_LIGAR = 30;
const int LIMIAR_DESLIGAR = 15;

const unsigned long TEMPO_MAX_LIGADA = 10UL * 60UL * 1000UL;

unsigned long inicioBomba = 0;

Cada linha possui uma finalidade específica. Vamos estudá-las individualmente.

O que é uma variável?

Uma variável é um espaço reservado na memória do computador para guardar um valor. Podemos imaginar a memória como um conjunto de pequenas caixas. Cada caixa possui um nome e guarda uma informação. 


Memória RAM

├────────────────────┤

Valor inicial false

Primeira variável 



bool bombaLigada = false;

Ela informa se a bomba está ligada.

Valor Significado
true Bomba ligada.

Por que usar bool?

Porque existem apenas duas possibilidades.

O tipo bool foi criado exatamente para representar estados lógicos.

Segunda variável

Essa variável informa se o sistema entrou em modo de proteção.

Enquanto ela permanecer false, o sistema poderá funcionar normalmente. Se algum problema ocorrer, ela será alterada para true.

O que significa const? 



const int LIMIAR_LIGAR = 30;

A palavra const significa constante. Depois que o valor é definido, ele não pode mais ser alterado durante a execução do programa.

Por que usar constantes?

Imagine que o nível da caixa d'água deve ser alterado futuramente. Em vez de procurar dezenas de números espalhados pelo código, basta modificar apenas a constante. 



const int LIMIAR_LIGAR = 35;

Todo o restante do programa continuará funcionando.

LIMIAR_LIGAR

Representa a distância mínima para ligar a bomba.

Se o sensor medir 30 centímetros ou mais, significa que o nível da água está baixo.

LIMIAR_DESLIGAR

Representa o nível em que a bomba será desligada.

Quando a água subir até aproximadamente 15 centímetros, o sistema interrompe o bombeamento.

O que é unsigned long? 



unsigned long

É um tipo inteiro capaz de armazenar números muito grandes. Ele é utilizado porque a função millis() pode atingir valores elevados durante o funcionamento do Arduino.

O significado de UL

UL significa:

  • U → Unsigned
  • L → Long

Esses sufixos garantem que o cálculo seja realizado utilizando números grandes sem risco de perda de precisão.

inicioBomba 



unsigned long inicioBomba = 0;

Essa variável armazenará o instante em que a bomba foi ligada. Mais tarde ela será utilizada junto com a função millis() para descobrir há quanto tempo a bomba permanece funcionando.

Você sabia?

Embora constantes não possam ser alteradas pelo programa, elas ajudam o compilador a identificar possíveis erros antes mesmo da gravação no Arduino.
Erro comum

É muito comum iniciantes confundirem const com #define. Ambos definem valores fixos, mas const possui tipo definido e participa das verificações feitas pelo compilador, enquanto #define realiza apenas substituição de texto.

Resumo

  • unsigned long armazena grandes valores inteiros.
  • inicioBomba registrará quando a bomba foi ligada.
  • TEMPO_MAX_LIGADA representa 10 minutos em milissegundos.

Capítulo 7

A Inicialização do Sistema: setup()

Todo programa Arduino possui uma função especial chamada setup(). Ela é executada apenas uma única vez, logo após o Arduino ser ligado ou reiniciado. Sua principal responsabilidade é preparar todo o hardware antes que o programa comece a funcionar continuamente.

O código do nosso projeto 



void setup() {

    Serial.begin(9600);

    digitalWrite(RELAY_PIN, HIGH);

    pinMode(RELAY_PIN, OUTPUT);

}

Embora possua apenas três instruções, cada uma delas desempenha um papel fundamental no funcionamento do sistema.

A palavra void 



void setup()

A palavra void indica que essa função não retorna nenhum valor. Ela apenas executa ações.

Primeira instrução 



Serial.begin(9600);

Essa linha inicia a comunicação serial entre o Arduino e o computador.

O que é comunicação Serial?

É um canal de comunicação utilizado para enviar e receber informações.

↓

↓

Tudo que enviarmos utilizando Serial.print() aparecerá no Monitor Serial da IDE.

Por que 9600?

O número 9600 representa a velocidade da comunicação. Sua unidade é bits por segundo (bps).

Velocidade Uso comum
9600 Projetos simples
57600 Projetos intermediários
115200 Grande volume de dados

Segunda instrução 



digitalWrite(RELAY_PIN, HIGH);

Antes mesmo de configurar o pino como saída, o programa define o estado lógico HIGH. Isso faz com que o relé permaneça desligado logo na inicialização.

Por que desligar primeiro?

Imagine que a bomba ligasse imediatamente ao energizar o Arduino. Isso poderia causar funcionamento inesperado e até danos ao equipamento. Por isso, primeiro garantimos um estado seguro.

Terceira instrução 



pinMode(RELAY_PIN, OUTPUT);

Agora informamos ao microcontrolador que o pino do relé será utilizado como saída.

O que é pinMode()?

Cada pino digital do Arduino pode assumir duas funções:

  • INPUT → Receber sinais.
  • OUTPUT → Enviar sinais.

Como queremos controlar um relé, utilizamos OUTPUT.

Por que configurar o pino?

Quando o Arduino liga, todos os pinos começam como entrada (INPUT). Isso evita conflitos elétricos. Somente depois informamos quais pinos serão utilizados como saída.

Analogia

Imagine um funcionário iniciando o expediente.

↓

↓

↓

O setup() faz exatamente essa preparação.

Você sabia?

Mesmo programas extremamente complexos normalmente possuem apenas um setup(). Toda a configuração inicial do hardware é feita nele.
Erro comum

Esquecer de configurar um pino como OUTPUT pode fazer com que o relé nunca seja acionado corretamente. Sempre configure o modo do pino antes de utilizá-lo.

Resumo

  • setup() é executado apenas uma vez.
  • Serial.begin() inicia a comunicação com o computador.
  • digitalWrite(RELAY_PIN, HIGH) mantém a bomba desligada na inicialização.
  • pinMode() define a função de cada pino.
  • Após o setup(), o Arduino entra no loop() para executar continuamente o programa.

Capítulo 8

A Leitura do Sensor Ultrassônico

Após finalizar toda a inicialização do hardware no setup(), o Arduino entra na função loop(). É nela que ocorre toda a lógica do sistema. A primeira instrução executada é:

O tipo long

A distância medida pelo sensor é armazenada em uma variável do tipo long. Esse tipo permite armazenar números inteiros maiores que um int, oferecendo maior segurança para cálculos.

O objeto sensor

Agora utilizamos esse objeto para realizar uma medição.

O sensor emite uma onda ultrassônica. Essa onda percorre o ambiente até encontrar um obstáculo. Quando bate na superfície da água, retorna ao sensor.

O método read() aceita diferentes unidades. No nosso projeto utilizamos centímetros.

Assim o valor retornado já vem convertido para centímetros.

Boa parte dos sistemas industriais valida os dados recebidos antes de tomar qualquer decisão. Essa prática aumenta a confiabilidade e reduz o risco de falhas.
Erro comum

Não confunda o operador lógico && (E) com o operador || (OU). No nosso projeto, todas as condições precisam ser verdadeiras ao mesmo tempo.

Resumo

  • if executa código apenas quando a condição é verdadeira.
  • cm > 0 valida a leitura.
  • cm < 400 garante que a distância está dentro da faixa do sensor.
  • && exige que todas as condições sejam verdadeiras.
  • ! inverte um valor booleano.
  • O sistema só continua se não houver falha de segurança.

Capítulo 10

Controlando a Bomba d'Água

Depois de validar que a leitura do sensor é confiável, o Arduino decide se deve ligar ou desligar a bomba. Essa decisão é baseada na distância medida pelo sensor e nos limites definidos anteriormente.

O código responsável por ligar a bomba 



if (!bombaLigada && cm >= LIMIAR_LIGAR) {

    bombaLigada = true;

    inicioBomba = millis();

    digitalWrite(RELAY_PIN, LOW);

}

Primeira condição 



!bombaLigada

O símbolo ! significa negação. Essa condição verifica se a bomba ainda está desligada.

Valor de bombaLigada Resultado de !bombaLigada
false true
true false

Assim evitamos tentar ligar uma bomba que já está ligada.

Segunda condição 



cm >= LIMIAR_LIGAR

Agora verificamos se a distância medida é maior ou igual ao limite definido para ligar a bomba.

No nosso projeto: 



const int LIMIAR_LIGAR = 30;

Portanto, quando o sensor medir 30 cm ou mais, significa que o nível da água está baixo.

Ligando a bomba 



bombaLigada = true;

Após decidir ligar a bomba, atualizamos a variável de estado. Ela passa a indicar que a bomba está funcionando.

Registrando o horário 



inicioBomba = millis();

A função millis() retorna o tempo, em milissegundos, desde que o Arduino foi ligado. Esse valor é armazenado para que, posteriormente, possamos calcular quanto tempo a bomba permaneceu ligada.

Acionando o relé 



digitalWrite(RELAY_PIN, LOW);

Nesse projeto o módulo relé trabalha com lógica invertida. Isso significa:

Sinal enviado Estado do relé
LOW Relé ligado
HIGH Relé desligado

Ao enviar LOW para o pino do relé, a bomba é energizada.

Desligando a bomba 



else if (bombaLigada && cm <= LIMIAR_DESLIGAR) {

    bombaLigada = false;

    digitalWrite(RELAY_PIN, HIGH);

}

O que significa else if?

O else if representa uma segunda decisão. Ela só será analisada caso a condição anterior seja falsa.

Primeira condição 



bombaLigada

Agora queremos garantir que a bomba realmente esteja ligada. Não faz sentido tentar desligar uma bomba que já está desligada.

Segunda condição 



cm <= LIMIAR_DESLIGAR

O sensor verifica se a água já atingiu o nível desejado. No projeto: 



const int LIMIAR_DESLIGAR = 15;

Quando a distância chega a 15 cm ou menos, a bomba é desligada.

Desligando o relé 



digitalWrite(RELAY_PIN, HIGH);

Como o relé utiliza lógica invertida, enviar HIGH interrompe a alimentação da bomba.

Você sabia?

Utilizar dois limites diferentes (30 cm para ligar e 15 cm para desligar) evita que a bomba fique ligando e desligando rapidamente quando o nível da água oscila próximo de um único valor. Esse comportamento é conhecido como histerese.
Erro comum

Se o mesmo limite fosse usado para ligar e desligar a bomba, pequenas oscilações na leitura do sensor poderiam fazer o relé comutar repetidamente, reduzindo sua vida útil.

Resumo

  • O sistema liga a bomba apenas quando ela está desligada e o nível da água está baixo.
  • O horário da partida é armazenado com millis().
  • O relé é acionado utilizando lógica invertida.
  • A bomba é desligada quando a água atinge o nível desejado.
  • Os dois limites implementam histerese, tornando o sistema mais estável.

Capítulo 11

Segurança, Comunicação Serial e Conclusão do Sistema

Além de controlar automaticamente a bomba d'água, o programa também possui mecanismos de proteção para evitar danos ao equipamento. Esses mecanismos tornam o sistema mais confiável e seguro.

Proteção por tempo máximo 



if (bombaLigada &&
    (millis() - inicioBomba >= TEMPO_MAX_LIGADA)) {

    bombaLigada = false;

    falhaSeguranca = true;

    digitalWrite(RELAY_PIN, HIGH);

}

Esse trecho verifica há quanto tempo a bomba permanece ligada. Se ultrapassar o tempo máximo definido, o sistema entra em modo de segurança.

Como funciona millis()

A função millis() retorna o número de milissegundos desde que o Arduino foi ligado.

Ao subtrair o instante atual do instante em que a bomba foi ligada, descobrimos quanto tempo ela está funcionando.

Falha de segurança 



falhaSeguranca = true;

Quando essa variável passa para true, o programa deixa de controlar a bomba automaticamente até que a falha seja corrigida e o sistema seja reiniciado.

Envio das informações pela Serial

Após tomar todas as decisões, o Arduino envia informações ao computador utilizando a comunicação Serial. Esses dados podem ser visualizados no Monitor Serial ou utilizados por outros programas.
